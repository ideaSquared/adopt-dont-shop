# @adopt-dont-shop/lib.notifications

## Purpose

Multi-channel notification client for the frontend apps — send, schedule, list,
and acknowledge notifications across email, push, in-app, and SMS channels, plus
preferences and templates.

## Location in the architecture

See [`docs/README.md`](../../docs/README.md#libraries) for where the shared
libraries sit. Reaches the gateway's `/api/v1/notifications/*` surface over
[`service.notifications`](../../services/notifications/README.md); live in-app
delivery arrives over the Socket.IO edge.

## Public API / exports

The canonical list lives in [`src/index.ts`](src/index.ts):

- `NotificationsService` — class client. Sending: `sendNotification`,
  `sendBulkNotifications`, `scheduleNotification`. Listing:
  `getUserNotifications`, `getNotification`. Acknowledging: `markAsRead`,
  `markAllAsRead`, `deleteNotification`. Preferences: `getUserPreferences`,
  `updatePreferences`, `setDoNotDisturb`. Templates: `getTemplates`,
  `processTemplate`. Config: `getConfig` / `updateConfig`.
- Types: `NotificationsServiceConfig`, plus the event / preference / template
  types re-exported from `./types`.
