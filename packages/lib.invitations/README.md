# @adopt-dont-shop/lib.invitations

## Purpose

Staff / volunteer invitation client for the frontend apps — send, list, cancel,
and accept invitations to rescue organisations.

## Location in the architecture

See [`docs/README.md`](../../docs/README.md#libraries) for where the shared
libraries sit. Reaches the gateway's `/api/v1/invitations/*` +
`/api/v1/staff/*` surface over [`service.rescue`](../../services/rescue/README.md);
the accept flow additionally provisions the invited user via `service.auth`.

## Public API / exports

The canonical list lives in [`src/index.ts`](src/index.ts):

- `InvitationsService` — class client: `sendInvitation`,
  `getPendingInvitations`, `cancelInvitation`, `getInvitationDetails(token)`
  (the accept page), `acceptInvitation(payload)`, `healthCheck`.
- Types: `InvitationsServiceConfig`, plus the invitation / pending / accept
  types re-exported from `./types`.
