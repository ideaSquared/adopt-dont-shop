# @adopt-dont-shop/lib.feature-flags

## Purpose

Typed hooks and constants for Statsig feature-flag integration across the apps.
Provides React hooks layered on `@statsig/react-bindings`, the known-gate /
known-config constants, and type-safe config shapes. This library no longer
contains a backend flag service — flags are managed through Statsig
(`@statsig/react-bindings` in apps, `statsig-node` in services).

## Location in the architecture

See [`docs/README.md`](../../docs/README.md#libraries) for where the shared
libraries sit. Apps consume `KNOWN_GATES` / `KNOWN_CONFIGS` alongside the
Statsig bindings; the frontend integration patterns are documented in
[`apps/client/src/docs/STATSIG_FEATURE_FLAGS.md`](../../apps/client/src/docs/STATSIG_FEATURE_FLAGS.md).
For the broader gating conventions see the `feature-flags` guidance.

## Public API / exports

The canonical list lives in [`src/index.ts`](src/index.ts):

- Hooks: `useFeatureGate`, `useDynamicConfig`, `useConfigValue` (on top of
  `@statsig/react-bindings`).
- Constants: `KNOWN_GATES` (e.g. `ENABLE_REAL_TIME_MESSAGING`,
  `ENABLE_ADVANCED_SEARCH`, `UI_SHOW_BETA_FEATURES`, …) and `KNOWN_CONFIGS`
  (`APPLICATION_SETTINGS`, `SYSTEM_SETTINGS`, `MODERATION_SETTINGS`).
- Types: the strongly-typed dynamic-config shapes (e.g.
  `ApplicationSettingsConfig`).

## Environment variables consumed

None read directly — services that evaluate gates server-side initialise
`statsig-node` with `STATSIG_SERVER_SECRET_KEY`, and apps configure the client
SDK. See [`docs/env-reference.md`](../../docs/env-reference.md) for the full
list.
