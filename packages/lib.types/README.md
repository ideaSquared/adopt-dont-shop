# @adopt-dont-shop/lib.types

## Purpose

Shared TypeScript types, permission constants, and default configurations — the
single source of truth for all permission-related types and default configs.
**Zero dependencies**, so it is safe to import from both frontend and backend
(the backend gets type definitions without pulling in any frontend code).

## Location in the architecture

See [`docs/README.md`](../../docs/README.md#libraries) for where the shared
libraries sit. This is the dependency-free root of the permission model:
`lib.permissions` re-exports everything here and adds the frontend service
classes (depending on `lib.api`), while `service.auth` / `service.gateway`
import the types + default-config functions directly with **no** frontend deps.
It is the most widely-consumed package in the workspace.

## Public API / exports

The canonical list lives in [`src/index.ts`](src/index.ts):

- **RBAC types**: `UserRole`, `Permission`, `PermissionAction`,
  `PermissionResource`, `UserWithPermissions`, `PermissionsServiceConfig`, …
- **Field-level permission types**: `FieldAccessLevel`,
  `FieldPermissionResource`, `FieldPermissionConfig`, `FieldAccessMap`, …
- **Rescue permission constants**: `RescuePermissions`,
  `RescuePermissionGroups`, and the individual constants (`STAFF_CREATE`,
  `RESCUE_SETTINGS_UPDATE`, …).
- **Default configs**: `defaultFieldPermissions`, `getDefaultFieldAccess`,
  `getFieldAccessMap`.
