# @adopt-dont-shop/lib.permissions

## Purpose

Frontend permission services for the apps. Re-exports every type from
`@adopt-dont-shop/lib.types` and adds the frontend-only, API-backed service
classes (`PermissionsService` for RBAC, `FieldPermissionsService` for
field-level access) that depend on `lib.api`. Consumed by `app.admin`,
`app.client`, and `app.rescue`.

## Location in the architecture

See [`docs/README.md`](../../docs/README.md#libraries) for where the shared
libraries sit. `lib.types` is the source of truth (pure types + default
configs, zero deps — safe for frontend and backend); this package is the
frontend layer on top, depending on `lib.api`. **Services never import from
`lib.permissions` or `lib.api`** — they import defaults from `lib.types`
directly. `service.auth` owns the `field_permissions` table and resolves
effective access over gRPC; `service.gateway` exposes the admin REST surface at
`/api/v1/field-permissions/*`. Related backend action-level checks live in
[`@adopt-dont-shop/authz`](../authz/README.md).

## Public API / exports

The canonical list lives in [`src/index.ts`](src/index.ts):

- **Types** (re-exported from `lib.types`): the RBAC set (`UserRole`,
  `Permission`, `PermissionAction`, `PermissionResource`,
  `UserWithPermissions`, …) and the field-level set (`FieldAccessLevel`,
  `FieldAccessMap`, `FieldPermissionRule`, `FieldPermissionCheckResult`, …).
- **Services**: `PermissionsService` (RBAC: `hasPermission` /
  `hasAnyPermission` / `hasAllPermissions` / `hasRole` / `getUserPermissions` +
  admin `assignRole` / `grantPermissions` / `revokePermissions`, 5-minute
  cache) and `FieldPermissionsService` (`getFieldAccess`,
  `getEffectiveFieldAccessMap`, `maskFields` / `maskFieldsArray`,
  `getWriteBlockedFields`, admin `updateFieldPermission` /
  `deleteFieldPermission`, 2-minute cache).
- **Defaults**: `defaultFieldPermissions`, `getDefaultFieldAccess`,
  `getFieldAccessMap`.
- **Rescue constants**: `RescuePermissions`, `RescuePermissionGroups`.

Field permissions are two-layer — `effective = override ?? default ?? 'none'`
— with the defaults (`src/config/field-permission-defaults.ts`) as the security
floor. Security invariants baked into the defaults: `password` / `resetToken` /
`twoFactorSecret` / `backupCodes` are always `none`; adopters can't see
`interview_notes` / `home_visit_notes` / `score` on applications or
`medical_notes` / `microchip_id` on pets. Field keys match each resource's
serialized shape (users/rescues camelCase, pets/applications snake_case).

## Environment variables consumed

None — the services take config (`{ debug }`) via constructor args and reach
the backend through the shared `lib.api` client.
