# @adopt-dont-shop/lib.utils

## Purpose

Shared utility library — 25+ common functions grouped into five categories:
date & time (formatting, relative time, parsing, business-hours checks), string
(slugify, truncate, sanitise, id generation), data transformation (deep clone,
merge, flatten/unflatten, array→map), validation (email, phone, URL, file type,
each returning a detailed result), and formatting (currency, file size, phone,
address). Exposed as a configurable `UtilsService` used across the apps.

## Location in the architecture

See [`docs/README.md`](../../docs/README.md#libraries) for where the shared
libraries sit. This is a leaf utility library with minimal dependencies,
consumed by the apps and other libraries.

## Public API / exports

The canonical list lives in [`src/index.ts`](src/index.ts). `UtilsService` is
constructed with `{ debug?, timezone?, currency? }` (with `updateConfig` /
`getConfig` at runtime) and exposes:

- **Date & time**: `formatDate`, `formatRelativeTime`, `parseDate`,
  `isBusinessHours`.
- **String**: `slugify`, `truncate`, `sanitizeInput`, `generateId`,
  `capitalizeWords`.
- **Data transformation**: `deepClone`, `mergeObjects`, `flattenObject`,
  `unflattenObject`, `arrayToMap`.
- **Validation**: `isValidEmail`, `isValidPhone`, `isValidURL`,
  `validateFileType` (each returns a `ValidationResult`).
- **Formatting**: `formatCurrency`, `formatFileSize`, `formatPhoneNumber`,
  `formatAddress`.

Plus the supporting types (`DateFormatOptions`, `TruncateOptions`,
`SlugOptions`, `ValidationResult`, `CurrencyOptions`, `AddressData`, …). For
UK-locale conventions on dates / phone / currency, see the `uk-localization`
guidance.
