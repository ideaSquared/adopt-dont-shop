# @adopt-dont-shop/lib.validation

## Purpose

Canonical Zod schemas for the User, Pet, Rescue, and Application domains — one
source of truth for entity-shaped data. There's no runtime service surface; all
exports are pure Zod schemas and their inferred types. Used by the gateway +
gRPC services for request validation, by per-service domain checks before
persisting to Postgres, and by frontend forms (e.g. `lib.auth`).

## Location in the architecture

See [`docs/README.md`](../../docs/README.md#libraries) for where the shared
libraries sit. The schemas deliberately track each owning service's Postgres
schema (`services/<name>/src/migrations/`) — eliminating drift between the
validation layer and the DB is exactly why this library exists.

## Public API / exports

The canonical list lives in [`src/index.ts`](src/index.ts); schema modules under
[`src/schemas/`](src/schemas):

- **User**: `UserStatusSchema`, `UserTypeSchema`, plus the request shapes used
  by `lib.auth`.
- **Pet**: `PetCreateRequestSchema`, `PetUpdateRequestSchema`,
  `PetStatusUpdateRequestSchema`, `PetSearchFiltersSchema`,
  `BulkPetOperationRequestSchema`, plus the enum schemas (`PetStatusSchema`,
  `PetTypeSchema`, `GenderSchema`, `SizeSchema`, `AgeGroupSchema`, …).
- **Rescue** and **Application**: request / search / update shapes.

Each schema also exports its `z.infer<>` type (e.g. `PetCreateRequest`,
`PetSearchFilters`).
