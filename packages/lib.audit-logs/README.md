# @adopt-dont-shop/lib.audit-logs

## Purpose

Frontend audit-log access for the admin app — type-safe reads of audit-log data
with filtering, pagination, and date-range support, backed by the gateway's
`/api/v1/audit/*` surface (which proxies to `service.audit`).

libraries sit. Consumed by `app.admin`; the read path terminates at
[`service.audit`](../../services/audit/README.md) via the gateway.

## Public API / exports

The canonical list lives in [`src/index.ts`](src/index.ts):

- `AuditLogsService.getAuditLogs(filters?)` → `{ data, pagination }`. Filters:
  `action`, `userId`, `entity`, `level`, `status`, `startDate` / `endDate`
  (defaults to the last 7 days), `page`, `limit` (default 50). Results are
  sorted newest-first.
- Types + enums: `AuditLog`, `AuditLogLevel` (`INFO` / `WARNING` / `ERROR`),
  `AuditLogStatus` (`success` / `failure`).
